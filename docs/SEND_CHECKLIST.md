# HST-OUTREACH-01 — Send Checklist

> Complete every item before sending any outreach message.

---

## Prototype Readiness

- [ ] Prototype is deployed on Vercel (or alternative hosting)
- [ ] All 18 routes load without errors
- [ ] Homepage renders correctly
- [ ] `/demo` page loads and displays all sections
- [ ] `/pilot` page loads and displays all sections
- [ ] `/pilot/value` page loads and displays correctly
- [ ] `/pilot/roadmap` page loads and displays correctly
- [ ] `/pilot/story` page loads and displays correctly
- [ ] `/journey` page renders correctly
- [ ] `/compare` page renders correctly
- [ ] `/trust-safety` page renders correctly
- [ ] `/request` form loads and works (client-side)
- [ ] `/contact` form loads and works (client-side)
- [ ] No broken images or missing assets
- [ ] All internal navigation links work
- [ ] Build passes with 0 errors (`npm run build`)

---

## Content Safety Review

- [ ] No false partnership claims anywhere on the prototype
- [ ] No Cereneo logo or name used
- [ ] No specific provider names or logos used without permission
- [ ] No medical prediction language ("will improve", "guarantees recovery")
- [ ] No diagnosis claims ("AI detects", "identifies conditions")
- [ ] No instant booking claims ("book now", "reserve your stay")
- [ ] All disclaimers present on relevant pages
- [ ] "For discussion purposes only" language included
- [ ] "Independent concept" language used appropriately
- [ ] No patient data stored or referenced
- [ ] No pricing for medical procedures (only package concepts)
- [ ] No protected Swiss cross logos or official marks

---

## Email Readiness

- [ ] Email draft reviewed for spelling and grammar
- [ ] Recipient name correctly spelled
- [ ] Recipient title and organisation correct
- [ ] Subject line chosen and reviewed
- [ ] Email does not sound like a generic template
- [ ] No internal project references ("HST-MVP", "phase 2", etc.)
- [ ] Meeting time ask is clear and short (20–30 minutes)
- [ ] No pressure language
- [ ] No false claims
- [ ] "Independent concept" or similar language included
- [ ] Contact details correct (name, optional phone/LinkedIn)

---

## Recipient Verification

- [ ] Correct person identified (decision-maker or relevant department)
- [ ] Organisation is a genuine Swiss neuro-health provider
- [ ] No competitors contacted by mistake
- [ ] LinkedIn profile verified (if using LinkedIn)
- [ ] Email address verified (not guessing)
- [ ] No mass CC or BCC
- [ ] Personalised opening (not "Dear Sir/Madam")

---

## Follow-up Preparedness

- [ ] Follow-up email drafts ready
- [ ] First follow-up scheduled for 5–7 days after initial send
- [ ] Second follow-up content prepared (if needed)
- [ ] Meeting agenda prepared
- [ ] Demo script reviewed
- [ ] Objection-handling document accessible
- [ ] Demo links prepared (with placeholders for deployed URL)
- [ ] Thank-you email template ready

---

## Before Hitting Send

- [ ] Read the email aloud to check tone
- [ ] Check all placeholder brackets `[Name]` are replaced
- [ ] Confirm attachment (if any) is correct
- [ ] Send a test to yourself first
- [ ] Verify the email arrives and renders correctly on mobile
- [ ] Confirm the "from" name is clear and professional
- [ ] No tracking pixels or read receipts (optional — avoid if uncertain)
- [ ] Calendar invite ready if booking a specific time

---

## Final Verification

- [ ] `npm run build` passes with 0 errors on the deployed version
- [ ] All demo links use the deployed URL, not localhost
- [ ] `robots.txt` allows/disallows as intended (presentation pages are noindex)
- [ ] You are comfortable answering the questions in the Objection Handling document

---

*Checking every item on this list protects your credibility and ensures the conversation starts on the right foot. Do not skip any item.*
