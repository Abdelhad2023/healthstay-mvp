## Instructions Before Sending

1. **Replace `LIVE_URL_PLACEHOLDER`** with the actual Vercel deployment URL (e.g. `https://healthstay.vercel.app`) before sending.

2. **Test every link** before sending. Walk through each page to confirm:
   - Pages load without errors
   - Images and content render correctly
   - All internal navigation works
   - No placeholder or broken content appears

3. **Do not include links** in initial LinkedIn messages — only send after a connection request is accepted or after a meeting.

4. **Include context** with each link — do not send a bare list. Example:
   > "The pilot overview page explains the full concept: /pilot
   > The controlled demo page is designed for a 5-minute walkthrough: /demo
   > The provider value page summarises benefits for your team: /pilot/value"

5. **If the prototype has been updated** between the meeting and sending links, note what changed.
